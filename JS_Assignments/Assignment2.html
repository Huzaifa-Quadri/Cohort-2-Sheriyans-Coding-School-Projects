Q2. Variable Hoisting in JavaScript
a. Predict output of: console.log(a); var a = 10
b. Predict output of: console.log(b); let b = 10
c. Predict output of: test() function test() { console.log(“Hello”) }
d. Try writing a function expression before initialization and call it: hello() var hello = function() { console.log(“Hi”) } Write what happened and why.
e. Write one sentence: What gets hoisted? What does not get hoisted fully?
Explnation:
1.(d) Function will be called normally as javascript hoists entire function definition at top of scope during compilation phase. This works because the JavaScript engine stores the complete function in memory before executing any code.
This is not true for function expressions.

1.(e). Function declarations are hoisted fully along with their full definitions and only the declaration of var variable gets hoisted at top of scope.
Function expression and let/const variables are not hoisted.
